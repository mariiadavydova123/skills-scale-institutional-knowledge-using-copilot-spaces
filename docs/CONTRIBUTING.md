# Contributing

Thanks for contributing! This repository follows a simple workflow to make contributions consistent and reviewable.

1. Fork the repo (if you don't have push rights) or create a branch from `main`.
   - Branch name format: `improve/<short-description>` or `fix/<short-description>`
2. Make your changes on a new branch.
3. Add or update docs under `docs/` when the change affects processes, user-visible behavior, or contributor workflows.
4. Run tests and linters locally.
5. Open a pull request against `main` and use the PR template. Reference the related issue (e.g. `#4`).
6. Request reviewers and address feedback.

If you're unsure about where to add a change or which process document to update, open a short issue describing the change and request guidance.
